Web Page creation

For the table above I created a skeleton html file in Atom to start with then added some preconnect links for fonts, added a title and linked to my stylesheet. I then started to try and create a table by hand but realized how much time it was going to take and found an online table generator to make the basic table structure. If you have an industry approved tool that saves time, then I believe it makes you more efficient! I then went through the code I got from that tool and fixed the spacing, added the text values, and chose the background colors for the header cells. This proved to be difficult as I couldn't find a way to get the Chrome color selector tool to grab color data from a .png image so I had to try to get the best color match by hand. I think the assignment colors are the old RRU scheme because I couldn't find a match on any current RRU pages.

I realized after creating the table that while my version has editable fields, it doesn't have working formulas to calculate the total percentages in the last column and bottom row but to do this properly in Javascript from scratch is something I am still learning

I thought that under normal circumstances if I was given a table of data to put on a web page, it would more than likely already include data and it would also likely come from Excel or an editable table from Word. So I created a working table with formulas by first making the table in Excel, adding in the correct calculations for each cell, column, and row, then converting the table to html and adding it to the web page for the table you see below.

HMTL best practices applied to online learning

  • Make sure the formatting, text size, and font choices maxmimize readability, especially for anyone who may have a visual impairment.
  • Make sure the choice of font colors and the background/graphics choices ensure a viable contrast level and are also useable in color blind modes (e.g. no colour).
  • Make sure there is a central page or location that is the only place where important dates are listed so when you need to update a time/date/location, you don't need to hunt down obscure locations to change the data.
  • Make sure that titles of all assignments, activities, and filenames are consistent throughout the course. This avoids confusion for students and also makes for fewer mistakes while coding the pages and links.
  • If your links are pointing to 3rd party provided content, you must check these links on a regular basis to make sure they still work. Best practice would be to host this material yourself on a separate server.
  • Make sure links show where they are actually pointing to and not obfuscated like bit.ly links.
  • Use external style sheets and not inline styles to improve formatting consistency.
  • Always include alt text for any images or links for use with screen readers.
  • Make sure that it is clear what is expected of students, e.g. what are your individual assignments and due dates, what are your team assignments etc.
